# Security Policy

## Reporting a vulnerability

If you believe you have found a security issue in Project Cosmos — for example a vulnerability in any code published in this repository, an exposure risk in the way the ontology or schemas are defined, or any concern about how this repository is operated — please report it to us privately by email:

**[info@cybercrime-atlas.org](mailto:info@cybercrime-atlas.org)**

Please include:

- A description of the issue and its potential impact
- Steps to reproduce, if applicable
- Any relevant supporting material (screenshots, logs, references)
- Whether you are willing to be credited in the eventual disclosure, and if so, how you would like to be named

Please **do not** open a public GitHub issue for security-sensitive reports.

## What to expect

- We will acknowledge your report within **5 business days** of receipt.
- We will provide an initial assessment of severity and likely remediation timeline within **15 business days**.
- We will keep you informed of progress, and we will not publicly disclose the issue until a fix or mitigation is in place — unless you ask us to do so sooner, or unless an active exploitation in the wild requires earlier notification.

## Scope

This policy covers:

- Code, scripts, and tooling published in this repository
- The ontology definitions and schemas, in cases where their structure could create downstream risk for adopters
- Configuration of this repository itself

Out of scope:

- Issues with third-party software that merely uses Project Cosmos
- Generic best-practice suggestions that are not vulnerabilities
- Reports about Cybercrime Atlas investigative work or other internal World Economic Forum systems — those should go through standard WEF / Cybercrime Atlas channels

## Responsible disclosure

We support coordinated disclosure. We commit to:

- Treating reports confidentially while we investigate
- Working with the reporter on disclosure timing
- Crediting the reporter in any eventual public advisory, if they wish

## Out-of-scope research

Please do **not**:

- Test against private Cybercrime Atlas infrastructure or investigation systems
- Attempt to access non-public data or accounts
- Conduct denial-of-service, social-engineering, or physical attacks

If in doubt, ask first at the address above.
